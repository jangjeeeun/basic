3.함수

  • 함수 : 프로그램의 가장 기본적인 단위
  • 함수의 크기 : 함수는 작게 만들수록 좋다.
  • 함수에서 들여쓰기는 1단이나 2단을 넘어서는 안 된다.
  • 함수는 한 가지를 해야 한다.
    • 추상화 수준이 하나인 단계만 수행
    • 여러 작업 중인 함수는 의미 있는 이름으로 다른 함수를 추출할 수 있다.
  • 내려가기 규칙 : 코드는 위에서 아래로 이야기처럼 읽혀야 좋다.
  • Switch문 : switch문은 작게 만들기 어렵다 -> 다형성 (Polymorphisim) 활용
  • 여러 단어가 쉽게 읽히는 명명법을 사용 + 일관성
    • 함수의 인수가 늘어 날 수록 코드 이해도가 떨어진다.
      • -> 인수는 3개 이하로
      -> 출력 인수 // 입력 인수보다 이해하기 어려움 // 입력으로 해석 // 거의 사용할 필요가 없다.
    • 인수를 넘기는 경우 (1항)
      • -> 인수에 질문을 던지는 경우 (예) boolean fileExists("MyFile");
      -> 인수를 뭔가로 변환해 결과를 반환하는 경우 (예) StringBuffer transform(StringBuffer in); // 변환 형태 유지
      -> 함수와 인수가 동사/명사 쌍을 이뤄야 함
    • 인수를 넘기는 경우 (2항 이상) -> 순서를 인위적으로 기억
      -> 인수를 독자적인 클래스 변수 선언
      -> List형 인수 하나로 취급

  • Side Effect를 일으키는 경우
    시간적인 결함(temporal coupling) / 순서 종속성 (order dependency)
  • 오류 코드보다 예외를 사용하라! -> Try/Catch 블록 뽑아 내기 -> throws Exception 처리
  • 중복을 없애고 반복하지 마라!
